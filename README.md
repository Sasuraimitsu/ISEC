# 伊勢志摩水産物輸出促進協議会 公式サイト

三重県志摩市｜伊勢志摩水産物輸出促進協議会（ISESHIMA SEAFOOD EXPORT COUNCIL）の公式サイトです。
GitHub Pages で公開されており、`main` ブランチにコミットすると数分後に自動反映されます。

公開URL：https://sasuraimitsu.github.io/ISEC/

---

## ファイル構成

```
├── index.html            サイト本体（通常の更新では編集不要）
├── style.css             デザイン（色・レイアウト）
├── script.js             動作（日英切替・お知らせ等の自動生成）
├── data/
│   ├── news.json         ★ お知らせのデータ（お知らせ更新はこのファイルだけ）
│   └── documents.json    ★ 事業計画・事業報告の一覧データ
└── docs/
    ├── 01_teikan.pdf             定款
    ├── 02_soshiki_kitei.pdf      組織規程
    ├── 03_keiri_kitei.pdf        経理規程
    └── P-260710_jigyokeikaku_r8.pdf  令和8年度 事業計画
```

## 1. お知らせの更新方法

**`data/news.json` だけを編集します。index.html は触りません。**

1. GitHub でリポジトリを開き、`data/news.json` をクリック
2. 鉛筆アイコン（Edit this file）をクリック
3. 配列の**いちばん上**に、次の形で1件追加（上にあるものほど新着として表示されます）

```json
{
  "date": "2026.09",
  "datetime": "2026-09",
  "tag_ja": "お知らせ",
  "tag_en": "Notice",
  "body_ja": "日本語の本文をここに書きます。",
  "body_en": "English text here."
},
```

4. 「Commit changes」を押す → 数分後にサイトへ反映

### 記入ルール
- `tag_ja` はこれまで「お知らせ」「事業」「予告」を使用（自由に増やせます）
- 英語（`tag_en` / `body_en`）は省略可。省略すると英語表示時も日本語のまま表示されます
- **各項目の間のカンマ `,` と括弧 `{ }` の対応を崩さないこと**（詳細は下記「注意事項」）

## 2. 事業計画・事業報告の追加方法

書類番号のルール：

| 種別 | 番号 | バッジ色 | 例 |
|------|------|----------|-----|
| 事業計画 | **P-YYMMDD** | 緑 | P-260710（令和8年7月10日承認の計画） |
| 事業報告 | **R-YYMMDD** | 青 | R-270710（令和9年7月10日承認の報告） |

YYMMDD ＝ **総会等で承認を受けた日**。一覧はこの日付の新しい順に自動で並びます。

### 手順
1. PDFファイルを用意し、番号を頭に付けた半角英数のファイル名にする
   （例：`R-270710_jigyohokoku_r8.pdf`）
2. `docs/` フォルダにアップロード（GitHubの「Add file → Upload files」）
3. `data/documents.json` に1件追加：

```json
{
  "no": "R-270710",
  "type": "report",
  "date": "2027.07.10",
  "title_ja": "令和8年度 事業報告",
  "title_en": "FY2026 Business Report",
  "file": "docs/R-270710_jigyohokoku_r8.pdf"
}
```

- `type` は `plan`（事業計画）か `report`（事業報告）
- `date` は表示用の承認日

### 規程（定款・組織規程・経理規程）の差し替え
改定があった場合は、**同じファイル名のまま** `docs/` の PDF を上書きアップロードしてください。ファイル名を変えなければ index.html の修正は不要です。

## 3. 未定のため非表示にしている項目

| 項目 | 場所 | 状態・対応方法 |
|------|------|----------------|
| **監事** | `index.html` 構成メンバー欄 | 未就任のため行ごとコメントアウト（`<!-- -->`）で非表示。就任が決まったらコメント内のひな型（監事／氏名／所属・略歴）を書き換えて `<!--` と `-->` を外す |
| **事業報告** | 公開資料セクション | まだ発行実績がないため P-260710（事業計画）のみ掲載。初回の報告書は上記2.の手順で追加 |

### あわせて確認しておく設定
- **問い合わせ先メールアドレス**：`script.js` の冒頭 `CONTACT_EMAIL` に設定。変更する場合はこの1行を書き換える

## 4. デザイン面の調整メモ

- トップの縦書き「御食国の食材を、／世界へ。」は2行の段違い表示。**下げ幅**は `style.css` の `.tategaki-shift { padding-top: 2.4em; }` の数値で調整
- 日英切替は `script.js` 内の辞書 `I18N_EN` で管理。HTML側の `data-i18n="キー"` と対応

---

## 5. 注意事項

- **JSONの構文ミスに注意**：カンマの付け忘れ・付けすぎ、括弧の欠落が1か所でもあると、お知らせ／資料一覧の**全体が表示されなくなります**。編集後は [JSONLint](https://jsonlint.com/) 等で構文チェックを推奨
- **ローカルでの確認について**：`index.html` をPCでダブルクリックして開くと、ブラウザの制約により、お知らせ・資料一覧が空欄になることがあります（GitHub Pages 上では正常に動作します）。確認は公開後のURLで行ってください
- 反映まで数分かかることがあります。反映されない場合はブラウザの再読み込み（Ctrl+F5）を試してください
